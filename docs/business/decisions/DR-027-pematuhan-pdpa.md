# DR-027: Adakah data pelajar tertakluk kepada PDPA (Akta Perlindungan Data Peribadi 2010)?

## Status
Diputuskan

## Soalan
Adakah sistem CFK HUB perlu mematuhi keperluan formal Akta Perlindungan Data Peribadi (PDPA) Malaysia?

## Konteks
PDPA memerlukan pematuhan tertentu seperti notis privasi, persetujuan pengumpulan data, hak akses subjek data, dan prosedur aduan. Pematuhan formal memerlukan dokumentasi dan proses tambahan.

## Keputusan
**PDPA asas** — notis privasi ringkas dimasukkan dalam Google Forms pendaftaran pelajar, dan data disimpan dengan selamat mengikut amalan terbaik.

## Sebab
Sistem menyimpan data peribadi kanak-kanak dan ibu bapa. Walaupun pematuhan PDPA penuh tidak praktikal untuk MVP, sekurang-kurangnya ibu bapa perlu dimaklumkan tentang penggunaan data mereka semasa mendaftar. Notis ringkas dalam Google Forms (DR-026) adalah cara paling mudah untuk melaksanakan ini.

## Kesan
- Google Forms pendaftaran perlu mengandungi notis privasi ringkas: "Maklumat yang dikumpul akan digunakan untuk tujuan pengurusan program Chess For Kids sahaja dan tidak akan dikongsi kepada pihak ketiga"
- Ibu bapa perlu tanda kotak persetujuan sebelum boleh submit borang
- Data disimpan dengan selamat di cloud (DR-017) dengan akses terhad kepada kakitangan sahaja
- Tiada dokumentasi PDPA formal atau prosedur aduan diperlukan pada peringkat ini
- Boleh dikemas kini kepada pematuhan penuh jika organisasi berkembang
